# 기능 명세: 메인 메뉴

> 관련 PRD: 3.1 메인 메뉴

## 1. 목적

콘솔 애플리케이션의 진입점. 하위 기능(시료 관리, 주문, 모니터링, 출고 처리, 생산 라인)으로 이동할 수 있는 선택 화면을 제공하고, 전체 시료에 대한 요약 정보를 함께 노출한다.

## 2. 화면 구성 (예시, 자유 변경 가능 — PRD 4장 비고)

```
==================================================
 반도체 시료 생산주문관리 시스템
==================================================
[요약]
 등록 시료 수 : 5
 총 재고 수량 : 132
 진행 중 주문 : RESERVED 3 / PRODUCING 2 / CONFIRMED 4

[메뉴]
 1. 시료 관리
 2. 주문 (접수 / 승인 / 거절)
 3. 모니터링
 4. 출고 처리
 5. 생산 라인
 0. 종료
--------------------------------------------------
선택 >
```

## 3. 요약 정보 항목

| 항목 | 산출 방법 |
|---|---|
| 등록 시료 수 | Sample 저장소의 전체 개수 |
| 총 재고 수량 | 모든 Sample 의 `stockQuantity` 합 |
| 상태별 주문 수 | `RESERVED`/`PRODUCING`/`CONFIRMED` 개수 (모니터링 3.5 와 동일 규칙, `REJECTED` 제외, `RELEASE` 는 종료 건이므로 요약에서는 생략 가능) |

## 4. 입력 처리 규칙

- 숫자 입력을 기본으로 하되, 정의되지 않은 값 입력 시 오류 메시지를 출력하고 메인 메뉴를 재표시한다.
- `0` 입력 시 프로그램을 정상 종료한다.
- 각 하위 메뉴 진입 후 하위 화면에서 복귀하면 항상 메인 메뉴 요약 정보를 최신 상태로 다시 계산하여 표시한다.

## 5. 예외 처리

- 입력값이 숫자가 아닌 경우 (예: 문자, 공백): 예외를 발생시키지 않고 "잘못된 입력입니다" 안내 후 재입력을 받는다.
- 시료가 하나도 등록되지 않은 초기 상태에서도 메인 메뉴는 정상 표시되어야 한다 (요약 수치는 0으로 표기).

## 6. 관련 컴포넌트 (MVC)

- **View**: `MainMenuView` — 메뉴 출력, 입력 수신
- **Controller**: `MainMenuController` (또는 `AppController`) — 메뉴 선택에 따라 하위 Controller 로 라우팅
- **Model**: 요약 정보 산출을 위해 `SampleRepository`, `OrderRepository` 조회 필요 (직접 상태 변경은 없음)
