# 기능 명세: 시료 관리

> 관련 PRD: 3.2 시료 관리
> 도메인 모델: [00-domain-model.md](00-domain-model.md) - Sample

## 1. 목적

시스템의 가장 기본 단위인 시료(Sample)를 등록/조회/검색한다. **시스템에 등록된 시료만 주문이 가능하다.**

## 2. 하위 기능

### 2.1 시료 등록

**입력 값**

| 항목 | 필수 | 검증 규칙 |
|---|---|---|
| 이름 (name) | Y | 공백/빈 문자열 불가, 기존 등록된 이름과 중복 불가 |
| 평균 생산시간 (avgProductionTime) | Y | 양수 |
| 수율 (yield) | Y | 0 초과 1 이하 실수 |
| 시료 ID (sampleId) | N (자동 채번 권장) | 자동 채번 시 사용자 입력 없음. 수동 입력을 허용할 경우 중복 불가 |

**처리 흐름**
1. 입력값 검증 (형식/범위/중복)
2. 검증 실패 시 오류 사유를 출력하고 등록을 취소 (재입력 유도 또는 메뉴 복귀는 구현 시 결정)
3. 검증 성공 시 `stockQuantity = 0` 으로 초기화하여 Sample 생성
4. 등록 완료 메시지 및 생성된 시료 정보(ID 포함) 출력

**엣지 케이스**
- 이름 중복 등록 시도 → 등록 거부, "이미 존재하는 시료 이름입니다" 안내
- 수율에 0 또는 음수 입력 → 등록 거부 ("생산이 불가능한 시료"가 되므로 방지)
- 평균 생산시간에 0 또는 음수 입력 → 등록 거부

### 2.2 시료 조회

**동작**
- 등록된 모든 시료의 목록을 표시한다.
- 각 시료의 **현재 재고 수량을 함께 표시**한다 (PRD 3.2 명시 요구사항).

**표시 항목 (예시)**

| sampleId | name | avgProductionTime | yield | stockQuantity |
|---|---|---|---|---|

**엣지 케이스**
- 등록된 시료가 없는 경우: "등록된 시료가 없습니다" 안내 후 목록은 빈 상태로 표시

### 2.3 시료 검색

**입력 값**
- 검색 키워드 (이름 기준, 부분 일치 검색 권장)

**동작**
- 이름에 검색어가 포함된 시료를 모두 조회하여 목록으로 표시 (조회 화면과 동일한 컬럼 구성)
- 대소문자 구분 여부는 구현 시 결정 (한글 위주이므로 큰 영향 없음)

**엣지 케이스**
- 검색 결과가 없는 경우: "검색 결과가 없습니다" 안내
- 빈 검색어 입력 시: 전체 목록을 반환하거나 재입력을 요구 (구현 시 결정, 후자 권장)

## 3. 관련 컴포넌트 (MVC)

- **Model**: `Sample`, `SampleRepository` (등록/조회/이름검색/재고갱신 인터페이스)
- **View**: `SampleView` — 등록 입력폼, 목록/검색 결과 출력
- **Controller**: `SampleController` — 입력 검증 후 Repository 호출, View 로 결과 전달

## 4. 다른 기능과의 연관성

- **주문 접수(3.3)**: 주문 생성 시 `sampleId` 유효성 검증에 `SampleRepository` 조회를 사용한다. 등록되지 않은 시료로는 주문 생성이 불가하다.
- **생산 라인(3.6)**: 생산 완료 시 해당 Sample 의 `stockQuantity` 를 증가시킨다.
- **출고 처리(3.7)**: 출고 시 해당 Sample 의 `stockQuantity` 를 감소시킨다.
