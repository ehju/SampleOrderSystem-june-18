# Phase 8 — 메인 메뉴 통합 & 콘솔 UX 마감

> 참조 명세 원문: [docs/specs/01-main-menu.md](docs/specs/01-main-menu.md)
> 코딩 컨벤션: [dev/coding-convention.md](dev/coding-convention.md)
> 이 문서는 자기완결적이다. 다른 phase 파일이나 PLAN.md를 참조하지 않고 이 문서만으로 작업한다.
> 전제: Phase 2~7 에서 시료 관리/주문 접수/승인·거절/생산 라인/모니터링/출고 처리 기능이 모두 구현되어 있다.

## 목표

모든 하위 기능을 메인 메뉴에서 라우팅하고, 전체 애플리케이션으로 통합한다.

## 구현 항목

### 1. `MainMenuView`/`MainMenuController`(또는 `AppController`)

5개 메뉴 + 종료:
1. 시료 관리
2. 주문 (접수 / 승인 / 거절)
3. 모니터링
4. 출고 처리
5. 생산 라인
0. 종료

**화면 구성 예시**
```
==================================================
 반도체 시료 생산주문관리 시스템
==================================================
[요약]
 등록 시료 수 : 5
 총 재고 수량 : 132
 진행 중 주문 : RESERVED 3 / PRODUCING 2 / CONFIRMED 4

[메뉴]
 1. 시료 관리
 2. 주문 (접수 / 승인 / 거절)
 3. 모니터링
 4. 출고 처리
 5. 생산 라인
 0. 종료
--------------------------------------------------
선택 >
```

### 2. 메인 메뉴 요약 정보 실시간 계산

| 항목 | 산출 방법 |
|---|---|
| 등록 시료 수 | `SampleRepository` 전체 개수 |
| 총 재고 수량 | 모든 `Sample` 의 `stockQuantity` 합 |
| 상태별 주문 수 | `RESERVED`/`PRODUCING`/`CONFIRMED` 개수 (`REJECTED` 제외, `RELEASE` 는 요약에서 생략 가능) |

### 3. 입력 처리 규칙

- 숫자 입력을 기본으로 하되, 정의되지 않은 값 입력 시 오류 메시지를 출력하고 메인 메뉴를 재표시한다.
- `0` 입력 시 프로그램을 정상 종료한다.
- 각 하위 메뉴 진입 후 복귀하면 항상 메인 메뉴 요약 정보를 최신 상태로 다시 계산하여 표시한다.

### 4. 공통 예외 처리

- 입력값이 숫자가 아닌 경우 (문자, 공백 등): 예외를 발생시키지 않고 "잘못된 입력입니다" 안내 후 재입력을 받는다.
- 범위 밖 메뉴 번호 입력 시에도 동일하게 안내 후 재입력을 받는다.
- 시료가 하나도 등록되지 않은 초기 상태에서도 메인 메뉴는 정상 표시되어야 한다 (요약 수치는 0으로 표기).

## 테스트 관점 (RED 단계에서 작성할 대상)

GoogleMock 으로 `SampleRepository`/`OrderRepository` 를 목 처리하여 요약 정보 계산과 라우팅 로직을 격리 테스트한다.

- 등록 시료 수 / 총 재고 수량 / 상태별 주문 수 요약 값이 Repository 데이터로부터 정확히 계산됨을 확인.
- 시료/주문이 하나도 없는 초기 상태에서 모든 요약 수치가 0으로 계산됨을 확인.
- 숫자가 아닌 입력, 정의되지 않은 메뉴 번호 입력 시 오류 안내 후 메뉴가 재표시됨을 확인 (예외를 던지지 않고 처리되는지 확인).
- `0` 입력 시 프로그램 종료 흐름으로 진입함을 확인.
- 각 메뉴 번호(1~5) 선택 시 해당 하위 Controller 로 라우팅됨을 확인.

## 완료 기준 (Definition of Done)

- [ ] 메인 메뉴에서 모든 하위 기능(1~5) 진입/복귀가 정상 동작한다.
- [ ] 전체 시나리오(시료 등록 → 주문 접수 → 승인(재고부족 케이스 유도) → 생산 완료 처리 → 출고 → 모니터링 확인)를 수동 시연하여 성공한다.
- [ ] 잘못된 입력(숫자 외 입력, 범위 밖 메뉴 번호)에 대한 공통 예외 처리가 테스트로 확인된다.
